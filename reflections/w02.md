This week I want to write about what a strategy pattern usage is in my LLM. I learned that it is an object oriented design pattern that allows me to define a family of algorithms and make them into thier own class. They are interchangeable at runtime so the application can use an interface and not know what algorithm is running in the background. The LLM backend wrapper referes to the code used to implament AI into then application. I leared that it is not usually ideal to hardcode API's into my buisiness logic, but it is better to make a wrapper to be the middleman which handles communication with my AI service. Ideally you use the strategy pattern inside the wrapper so that you can modilfy how you want the AI to behave without changing the core application. This will tie in nicely with our job pack assignment especially as we are suppposed to generate resumes or cover letters with multiple AI's. We can make a wrapper that will be able to handle these tasks with any LLM that it is given.
